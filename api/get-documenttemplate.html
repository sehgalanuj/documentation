GET - document_template

This endpoint retrieves the details of a specific document template.

Request

URL: {{url}}/documenttemplate?id={{template_id}}

HTTP Request Type: GET

Replace {{url}} with your specific DocQ domain. For example, if your domain is ndm.docq.app, replace {{url}} with https://ndm.docq.app/api.

Headers

Header Type Description
x-access-token String Access token for authentication (required)
Content-Type String Content type of the request, should be application/json (required)

Query Parameters

Parameter Type Description
id Integer Document Template ID (required)

cURL

curl -X GET "{{url}}/document_template?id=117" -H "x-access-token: {{access_token}}" -H "Content-Type: application/json"

Python (requests)

import requests

url = "{{url}}/document_template"
params = {
    "id": 117
}
headers = {
    "x-access-token": "{{access_token}}",
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers, params=params)
print(response.json())

Response Structure

Field Type Description
[] Array The root array contains all templates that match the query. If, for example, you have queried template ID 3, then the array response will have only one entry corresponding to the template ID 3, if this template exists.
[].id Integer Unique identifier of the document template.
[].name String Name of the document template
[].workflow_id Integer The workflow assigned to this template. The assigned workflow will be triggered if a form associated with the template is used to generate a document.
[].document_template_content_id Integer The default content access group assigned to a file created from this template.
[].document_tempalte_file_id Integer ID of the file used for the template.
[].filename String Name of the source file used for this template.
[].filetype String Currently limited to docx or pdf types.
[].priority String One of: urgent, high, normal, low, none.
[].fields Array List of fields included in the template. The JSON structure of each field within the template is dependent upon the type of fields. Here we will list the JSON fields that will always be part of the response.
[].fields[].id String ID of the field. This is the variable name used in the document.
[].fields[].label String Label of the field. This is just a visual descriptor of the field.
[].fields[].required Boolean Indicates if the field is mandatory.
[].fields[].type String This can be one of: text, email, number, date, boolean, textarea, dropdown, checkbox, customtext, currency, clause, signature.
[].groups Array An array of strings containing the group IDs that will be assigned to the document generated from this template.
