Introduction

This API documentation provides information about DocQ API endpoints that allows you to interact with the DocQ service to fetch a list of documents, get details of a specific document, download documents (or its individual file versions), and delete incomplete documents.

An important terminology to understand here is that document is essentially a container used by DocQ to bundle files, their versions and related data together. Within a document you may have several files which are basically individual versions that were created during the lifecycle of a document.

Another important thing to note is that once a document reaches the complete status, it is for all essential purposes locked. You may no longer manipulate this doucment under normal circumstances. One exception is a retention policy that would then allow for documents to be automatically purged after a certain amount of time.

Authentication

All API requests must include the x-access-token header with a valid access token. This token is used to authenticate and authorize your requests.

You can obtain this token by logging into the DocQ front-end for your instance, then navigating to Settings -> Plugins -> DocQ integration tokens and then clicking Add token to generate a new token.

Endpoints

Throughout this documentation, you will see the placeholder {{url}}. Replace {{url}} with your specific DocQ domain. For example, if your domain is ndm.docq.app, replace {{url}} with https://ndm.docq.app/api.

GET - document_pagination

This endpoint is used to fetch a paginated list of documents.

Request

URL: {{url}}/document_pagination?page_size=50&page=1&completed=false

HTTP Request Type: GET

Headers

Header Type Description
x-access-token String Access token for authentication (required)
Content-Type String Content type of the request, should be application/json (required)

Query Parameters

Parameter Type Description
page_size Integer Number of documents per page (required)
page Integer Page number (required)
completed Boolean Filter by completion status (required)
deleted Boolean Include deleted documents
search String Search query for documents
order String Order of documents (e.g., id,desc)
group_filter String Filter by group IDs
assign_to_user String Filter by assigned user
priority String Filter by priority
date_begin Date Filter by start date
date_end Date Filter by end date
archived Boolean Include archived documents
tags Integer Filter by tag ID
in_folder String Filter by folder

cURL

curl -X GET "{{url}}/document_pagination?page_size=50&page=1&completed=false" -H "x-access-token: {{access_token}}" -H "Content-Type: application/json"

Python (requests)

import requests

url = "{{url}}/document_pagination"
params = {
    "page_size": 50,
    "page": 1,
    "completed": False
}
headers = {
    "x-access-token": "{{access_token}}",
    "Content-Type": "application/json"
}

response = requests.get(url, headers=headers, params=params)
print(response.json())

documents[].group_name Array of Strings List of group names associated with the document
documents[].id Integer Unique identifier for the document
documents[].name String Name of the document
documents[].priority String Priority level of the document
documents[].source_name String Source of the document
documents[].tag_name Array List of tags associated with the document
documents[].workflow_name String Name of the workflow the document is part of
pending_docs Integer Number of pending documents
total_docs Integer Total number of documents

GET - document

This endpoint retrieves the details of a specific document.

Request

URL: {{url}}/document?id={{document_id}}

HTTP Request Type: GET

Headers

Header Type Description
x-access-token String Access token for authentication (required)
Content-Type String Content type of the request, should be application/json (required)

Query Parameters

Parameter Type Description
id Integer Document ID (required)
priority String Filter by priority (optional)
gdoc_id String Google Document ID (optional)

cURL

curl -X GET "{{url}}/document?id=3003" -H "x-access-token: {{access_token}}" -H "Content-Type: application/json"

GET - document-download

This endpoint downloads the specified document in binary format. You may use the document information (GET - document endpoint) to assign a filename, or create a custom one based on the details of the document itself. Use the file ID, along with the document ID, to download a specific version of the document.

Request

URL: {{url}}/document/download?id={{document_id}}

HTTP Request Type: GET

Headers

Header Type Description
x-access-token String Access token for authentication (required)

Query Parameters

Parameter Type Description
id Integer Document ID (required)
file_id String File ID (optional)

cURL

curl -X GET "{{url}}/document/download?id=644" -H "x-access-token: {{access_token}}"

Python (requests)

import requests

url = "{{url}}/api/document/download"
params = {
    "id": 644
}
headers = {
    "x-access-token": "{{access_token}}"
}

response = requests.get(url, headers=headers, params=params)
with open('document.pdf', 'wb') as file:
    file.write(response.content)

JavaScript (fetch)

fetch("{{url}}/api/document/download?id=644", {
    method: 'GET',
    headers: {
        'x-access-token': '{{access_token}}'
    }
})
.then(response => response.blob())
.then(blob => {
    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.style.display = 'none';
    a.href = url;
    a.download = 'document.pdf';
    document.body.appendChild(a);
    a.click();
    window.URL.revokeObjectURL(url);
});

DELETE - document

This endpoint deletes the specified document. It is important to note that only incomplete documents may be deleted using this endpoint. Once a document reaches the compled status, it is locked for security reasons and may never be deleted via the UI or API. This document may only be deleted by setting up a retention policy that will automatically delete documents after a certain period of time. You must speak with your account manager to get such a retention policy enabled.

Request

URL: {{url}}/document?id={{document_id}}

HTTP Request Type: DELETE

Headers

Header Type Description
x-access-token String Access token for authentication (required)
Content-Type String Content type of the request, should be application/json (required)

Query Parameters

Parameter Type Description
id Integer Document ID (required)

cURL

curl -X DELETE "{{url}}/document?id=17" -H "x-access-token: {{access_token}}" -H "Content-Type: application/json"

Python (requests)

import requests

url = "{{url}}/document"
params = {
    "id": 17
}
headers = {
    "x-access-token": "{{access_token}}",
    "Content-Type": "application/json"
}

response = requests.delete(url, headers=headers, params=params)
print(response.json())

JavaScript (fetch)

fetch("{{url}}/document?id=17", {
    method: 'DELETE',
    headers: {
        'x-access-token': '{{access_token}}',
        'Content-Type': 'application/json'
    }
})
.then(response => response.json())
.then(data => console.log(data));
